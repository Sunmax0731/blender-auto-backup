# v0.1.1 - Backup Preview

Release date: 2026-05-14

## Summary

This release adds `Preview Backup`, a dry-run workflow that checks the current backup settings before ZIP creation. Users can confirm the target file count, total byte count, and planned archive path without creating a destination folder, ZIP file, or `.partial` file.

## Changes

- Added `Preview Backup` to Scene Properties > Auto Backup.
- Added pure-Python `plan_backup` service logic so Preview uses the same source validation, destination exclusion, destination layout, and include / exclude glob rules as `Backup Now`.
- Updated user-facing README, installation, manual test, architecture, specification, test plan, release checklist, QCDS, and local Issue evidence for post-MVP release readiness.

## Install

1. Download `blender-auto-backup-0.1.1.zip` from the GitHub release assets.
2. In Blender, open `Edit > Preferences > Add-ons`.
3. Use `Install from Disk...` and select the ZIP.
4. Enable `Blender Auto Backup`.
5. Configure `Source Folder`, `Backup Folder`, and `Destination Layout` from Scene Properties > Auto Backup.
6. Press `Preview Backup` to confirm the backup target before running `Backup Now` or `Start Auto Backup`.

## Known Notes

- The add-on is file-system only and does not expose a web URL.
- Blender 4.2 or later is required.
- Blender 5.1.1 is the user-side runtime validation target for this release.
